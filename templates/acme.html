Using Certbot with LocalCA

Certbot is a popular ACME client that can be used with your LocalCA instance.

First, install the CA certificate on the client machine to ensure it trusts certificates issued by your LocalCA.
1. Install the CA certificate
# Download the CA certificate curl -k {{.BaseURL}}/download/ca -o ca.pem # For Debian/Ubuntu: sudo cp ca.pem /usr/local/share/ca-certificates/localca.crt sudo update-ca-certificates # For CentOS/RHEL: sudo cp ca.pem /etc/pki/ca-trust/source/anchors/localca.crt sudo update-ca-trust
2. Install Certbot
# Debian/Ubuntu: sudo apt-get update sudo apt-get install -y certbot # CentOS/RHEL: sudo dnf install -y certbot
3. Request a certificate
# For standalone verification (port 80 must be available): sudo certbot certonly --standalone \ --server {{.ACMEBaseURL}}/acme/directory \ --domain your-service.your-domain.local # For webroot verification: sudo certbot certonly --webroot \ --webroot-path /var/www/html \ --server {{.ACMEBaseURL}}/acme/directory \ --domain your-service.your-domain.local
4. Automatic renewal
# Test automatic renewal sudo certbot renew --dry-run # Certbot will automatically create a timer/cron job for renewals

Using acme.sh with LocalCA

acme.sh is a lightweight ACME client with low dependencies.

1. Install acme.sh
# Install acme.sh curl https://get.acme.sh | sh -s email=your-email@example.com
2. Set up the CA certificate
# Download the CA certificate curl -k {{.BaseURL}}/download/ca -o ca.pem # Tell acme.sh to use this CA export ACME_CA_CERT=`pwd`/ca.pem
3. Issue a certificate
# For HTTP verification ~/.acme.sh/acme.sh --issue \ --server {{.ACMEBaseURL}}/acme/directory \ -d your-service.your-domain.local \ -w /var/www/html # For DNS verification (add DNS record manually) ~/.acme.sh/acme.sh --issue \ --server {{.ACMEBaseURL}}/acme/directory \ -d your-service.your-domain.local \ --dns \ --yes-I-know-dns-manual-mode-enough-go-ahead-please
4. Install the certificate
# Install to Nginx ~/.acme.sh/acme.sh --install-cert -d your-service.your-domain.local \ --key-file /path/to/keyfile/in/nginx \ --fullchain-file /path/to/fullchain/in/nginx \ --reloadcmd "service nginx reload"

Using Traefik with LocalCA

Traefik is a modern reverse proxy and load balancer that can automatically manage certificates via ACME.

1. Set up your Traefik configuration
# docker-compose.yml example version: '3' services: traefik: image: traefik:v2.5 command: - "--providers.docker=true" - "--providers.docker.exposedbydefault=false" - "--entrypoints.web.address=:80" - "--entrypoints.websecure.address=:443" - "--certificatesresolvers.localca.acme.httpchallenge=true" - "--certificatesresolvers.localca.acme.httpchallenge.entrypoint=web" - "--certificatesresolvers.localca.acme.caserver={{.ACMEBaseURL}}/acme/directory" - "--certificatesresolvers.localca.acme.email=your-email@example.com" - "--certificatesresolvers.localca.acme.storage=/etc/traefik/acme.json" ports: - "80:80" - "443:443" volumes: - "/var/run/docker.sock:/var/run/docker.sock:ro" - "./ca.pem:/etc/ssl/certs/localca.pem:ro" - "./acme.json:/etc/traefik/acme.json" environment: - "LEGO_CA_CERTIFICATES=/etc/ssl/certs/localca.pem" - "SSL_CERT_FILE=/etc/ssl/certs/localca.pem" restart: unless-stopped whoami: image: traefik/whoami labels: - "traefik.enable=true" - "traefik.http.routers.whoami.rule=Host(`whoami.yourdomain.local`)" - "traefik.http.routers.whoami.entrypoints=websecure" - "traefik.http.routers.whoami.tls.certresolver=localca"
2. Download the CA certificate
# Download the CA certificate curl -k {{.BaseURL}}/download/ca -o ca.pem # Create an empty acme.json file with proper permissions touch acme.json chmod 600 acme.json
3. Start Traefik
docker-compose up -d

Using NGINX with LocalCA

You can use certbot or acme.sh with NGINX to automate certificate management.

1. Install NGINX and Certbot
# Debian/Ubuntu: sudo apt-get update sudo apt-get install -y nginx certbot python3-certbot-nginx # Download the CA certificate curl -k {{.BaseURL}}/download/ca -o ca.pem sudo cp ca.pem /usr/local/share/ca-certificates/localca.crt sudo update-ca-certificates
2. Configure NGINX
# Example server block server { listen 80; server_name your-service.your-domain.local; location / { # Your configuration here proxy_pass http://localhost:8080; } # Allow /.well-known/acme-challenge/ for HTTP-01 challenges location /.well-known/acme-challenge/ { root /var/www/html; } }
3. Request certificate with Certbot
sudo certbot --nginx \ --server {{.ACMEBaseURL}}/acme/directory \ -d your-service.your-domain.local
4. Verify automatic renewal
sudo certbot renew --dry-run
